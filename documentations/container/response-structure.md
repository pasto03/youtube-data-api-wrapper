# Container Response Structure

## Overview

This document describes the general structure of container responses. All containers follow similar patterns, with resource-specific variations.

## General Structure

All container items follow this base structure:

```python
@dataclass
class Item:
    kind: str              # Resource type (e.g., "youtube#channel")
    etag: str              # Entity tag
    id: str                # Resource ID
    snippet: Snippet       # Content details
    statistics: Statistics  # Metrics (if available)
    # ... other fields specific to resource type
```

## Snippet Structure

All snippets contain:

```python
@dataclass
class Snippet:
    title: str                    # Title/name
    description: str             # Description
    publishedAt: str             # Publish date (ISO 8601)
    thumbnails: list[ItemThumbnail]  # Thumbnail images
    # ... other fields specific to resource type
```

## Thumbnail Structure

All thumbnails follow this structure:

```python
@dataclass
class ItemThumbnail:
    quality: str      # "default", "medium", "high", "standard", "maxres"
    url: str          # Thumbnail URL
    width: str | None # Width in pixels
    height: str | None # Height in pixels
```

Common thumbnail qualities:

- `default` — Default thumbnail (120x90)
- `medium` — Medium quality (320x180)
- `high` — High quality (480x360)
- `standard` — Standard quality (640x480)
- `maxres` — Maximum resolution (1280x720 or higher)

## Statistics Structure

Statistics (when available) contain:

```python
@dataclass
class Statistics:
    viewCount: str | int        # View count

    subscriberCount: str | int  # Subscriber count (channels)
    videoCount: str | int      # Video count (channels)

    # ... other fields specific to resource type
```

### Accessing Snippet Data

